RoadMap (the future work)

If you have any other idea, think that some feature must be finish first than other or want to help in one of this items... join us in the mailing list or send me a mail.

This weeks I receive a lot of suggestion via mail, codeproject and mailing lists. This is the old roadmap, shortly I change te order and add the new features.

  • 1.4
    • Finish the TransformToRecord implementation
    • Some VB.NET examples
    • FieldQuoted can span multiple lines.
    • Add a DataStorages and DataLinks section to the Docs.
    • Add to the docs a page with the parameters of each ConverterKind.
    • An option to Discard Trailing Data or raise an error if found it.
    • An option to use a xls template for the ExcelStorage.
    • Make the data storages of SqlServer and Access that works without overrides and uses Attributes to get the names of the fields.
    • Progress Notification in the FileHelperEngine and DataStorages.
    • Add my XpProgressBar to the distribution for use in the previous topic.
  • 1.5
    • A RecordConvertAttribute to use a custom converter for the whole record and not field by field.
    • Conditional Records (to ignore records that contains certain substring or of the specified length
    • Diff Support, and option to compare two datastorages and generate a file with the differences.
    • Support for records that span multiple lines.
    • Support for multiple records (ie the FileHelperEngine can have more than one record class, with the same base, with a method to identify what to use.
    • Extend the demo project to show the rest of the Library
    • Better performance (internal buffers and forward caching)
    • Logging support via log4net or other lib.
    • Improved Code Smith templates.
  • 2.0
    • Nullable types of .NET 2.0.
    • FileHelpers Runner, a GUI for the library.
    • I don't know... your ideas are welcome !!!

If you find that there is a feature that I must include or you have a new idea (for the API, Source Code or Examples), only let me know it, sending an e-mail to marcosdotnet[at]yahoo.com.ar or entering My Home Page .
